Functions


What are they?
In programming, a function is basically a block of code that performs an operation and may return a value. Functions can have input parameters, which allow the caller to pass arguments to the function. Functions are one of the most powerful and reusable tools in programming. They allow you to group a set of instructions that perform a specific task, giving a name to that sequence of actions. The great benefit of using functions is the ability to call this "block of code" whenever needed, without having to write the same code multiple times. This way, functions help keep the code more organized, clear, and reusable.
A function typically receives inputs (known as parameters or arguments), executes a set of instructions, and then may return a value as a result. This structure allows us to pass different inputs to the function and get different results without changing the function's code itself. For example, a function that calculates the sum of two numbers can be called multiple times, passing different numbers, and it will give a different result each time.
To declare a function, we need to define its name, the parameters (if any), and what it should do. Once the function is defined, we can call it anywhere in the program to perform the task it was programmed to execute. This makes the code cleaner, as we can reuse functions whenever necessary instead of duplicating code.

An example of a function in algorithm
function Add(a, b) {
return a + b;
}
Here, this function receives two parameters, a and b, which are the numbers to be added. Inside the function, we add these two values and return the result. This function is a template to be used in any case... The rest of this code would be:
result = add(3, 4); In this case, the value 3 will be passed to parameter a and the value 4 will be passed to parameter b. The add function then uses these two values to calculate and return the sum of 3 and 4. The parentheses are the parameters and serve to link the inputs. In the function, it specifies that the inputs are a and b. And when the function is called (second code), the parentheses are the function inputs contextualized for that situation. Got it?

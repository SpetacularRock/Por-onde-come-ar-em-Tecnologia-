Programação Orientada a Objetos


"que nomezinho, ein..."

O que é?
Programação orientada a objetos é um paradigma de programação. O que quer dizer isso? Bem, signfica é um método/modo/jeito de programar. E neste jeito há novas regra e termos para aprender. A programação orientada a objetos(também conhecida como "POO") tem por objetivo facilitar e aproximar o programa da vida real, deixando o código com mais clareza e organização.


As 6 vantagens da Programação Orientada a Objetos



A POO destaca-se por ter 6 principais vantagens em relação aos outros paradigmas de programação: Confiável Ao alterar uma parte do código nenhuma outra é afetada. Oportuna Posso desenvolver partes do código paralelamente, de maneira paralela. Manutenível É mais fácil atualizar um software quando se está estruturado em POO. Extensível Não é estático, fácil de crescer. Reutilizável Pode-se facilmente aproveitar partes de um código em POO para criar outro. Natural É próximo da realidade, alto nível, próximo do programador, fácil de entender, natural.



Objetos e Classes


A programação orientada a objetos tem duas estruturas principais: os objetos e as classes. As classes são moldes e os objetos são os produtos. Vamos lá... Antes de se construir algo é necessário um planejamento, correto? Um desenho talvez, fichas, documentos. Mas em geral faz-se um planejamento do objeto que se pretende criar. Olhe em sua volta e selecione um objeto qualquer. Este objeto para o qual você está olhando precisou passar por um planejamento para que se pudesse enfim construir dezenas ou centenas iguais a este. A ideia permanece nestes dois elementos. Classes
As classes são estes planejamentos. Elas contém as informações características (atrbutos) e ações (métodos) que seus objetos virão a ter. É como se ela fosse uma forma de bolo. E o bolo seria... Objetos
Os objetos são as instâncias de uma classe, ou seja, são o resultado final. Se a classe é a forma, os objetos são os bolos. Com a mesma forma pode-se fazer bolos dos mais variados sabores e coberturas, certo? Contudo, todos eles terão o mesmo formato, pois forão todos confecionados na mesma forma. Assim são a relação entre os objetos e as classes.



Atributos e métodos



Os atributos são as caracteríticas destes objetos. E os métodos as ações relacionados a ele (que ele pode fazer). Novamente olhe em sua volta e escolha um objeto. Não, esse não. Escolha outro. Aí! Este mesmo. Perfeito, agora perceba que ele tem características tais como cor, peso, tamanho, altura, largura, marca, modelo, carga... Ah, uma coisa importante a se falar também é que características são diferentes de valores, ok? Duas canetas distintas possuem as mesmas características porque ambas possuem a característica "cor". Entretanto os valores/domínios de suas respectivas características são diferentes. Tenha isso em mente. Perfeito... agora este mesmo objeto que você está olhando contém ações (métodos). Um controle remoto por exemplo pode ligar, desligar, piscar, aumentar o volume, abaixar o volume... Uma caneta pode escrever, rabiscar, desenhar, pintar... ("Ah, mas não são a mesma coisa?" A resposta é depende. Depende do seu objetivo.). Portanto um objeto será perceptível pelos sentidos, uma vez que possui atributos e métodos, características e ações.




Até onde é objeto?




Bem... você pode até estar pensando: "Claro, né? Eu sei o que é um objeto. É facil de perceber." Bom, não duvido que você saiba o que é um objeto. Porém no contexto de programação orientada a objetos eles são um pouco mais abrangentes
Compromisso - Um compromisso é um objeto? Para responder a esta pergunta é necessário verficar se este elemento tem atributos e/ou métodos. Quais são as características de um compromisso? O horário, o dia, o local e o tipo (formal/informal), por exemplo. E seus métodos? O que se faz ou pode fazer com ele? Ele pode ser cancelado, adiado, antecipado ou remarcado. Portanto objetos são mais abrangentes e vão além da figura física e material.








Exemplo em algoritmo




Vamos criar a classe de um carro em algoritmo seguindo o paradigma da programação orientada ao objetos:
classe Carro {


caractere cor = "Preto";
caractere marca = "Fiat";
logico ligado = verdadeiro;


metodo ligar{
ligado = verdadeiro;
escreva ("O carro está ligado");
}
metodo acelerar() {
se (ligado = verdadeiro){
escreva ("Acelerando");
} senão escreva() ("Ligue o carro primeiro!");
}
metodo frear() {
se (ligado = verdadeiro){
escreva ("Freiando");
}
senão escreva ("Ligue o carro primeiro!");
}
}
}
Entendeu? Complicou um pouco mais? Vamos lá, primeiramente criamos a classe (molde). Depois definimos seus atributos (características) que no caso foram: cor, modelo, velocidade e ligado. Em seguida definimos alguns de seus métodos. As ações que o carro pode fazer. Para cada atributo definimos um valor. Colocamos texto nas variáveis do tipo caractere e um valor lógico na que era lógica (ligado). A variável lógica deste exemplo "ligado" representa se ele está ligado ou não (verdadeiro ou falso). É uma característica dele estar ligado ou não estar ligado. E essa característica é do tipo lógico. Pode estar ou não estar. E então definimos os métodos. A fins didáticos vou separar os métodos que fizemos em dois tipos: "o que muda um atributo" e "o que faz algo". O primeiro método que fizemos ele alterava o atributo ligado. Ou seja, ele ligava o carro. Portanto temos um atributo lógico que pode estar ou não estar e um método/ação que força ele a estar. Pegou a ideia? Ambos tem o mesmo nome porém representam aspectos diferentes (Um detalhe importante é que uma das dicas para visualmente diferenciar métodos de atributos é que usa-se "()"no final dos métodos.). E o outro tipo de método fazia uma ação. Neste momento você pode se perguntar: "Nossa, mas por que nos métodos há tantas coisas e nos atributos não???". A resposta é porque um atributo é sempre um atributo. Uma característica sempre vai ser uma característica independente da circunstância. Agora um método é muito mais variado. Precisa-se de vários fatores para uma ação ser realizada (você pode pensar no método "escrever", por exemplo. Onde a caneta precisa estar com carga, destampada e funcionando...). Portanto os métodos sempre serão mais preenchidos, pois eles precisam de circunstâncias mais específicas para acontecer. E no final colocamos um comando escreva em cada método para ver o que está acontecendo com o carro. Desta forma criamos a classe de um carro em POO.

Criando um objeto Para criar o objeto carro em algortimo simplesmente iremos fazer:
carro1 = Novo Carro;
Sempre iremos seguir a "fórmula": nome do objeto = novo Nome da Classe. É como se nós estivéssemos chamando um novo objeto para aquela classe, por isso a palavra nova (na verdade é exatamente isso.). E caso queira mudar algum atributo deste objeto ou chamar algum método:
carro1.cor = "Azul";
1.acelerar();
 Este objeto chamado carro1 agora é azul e está acelerando.


carro2.cor = "Cinza";
carro2.freando();
Já este outro objeto chamado carro2 é cinza e está freando.




Resumindo...



Programação Orientada a Objetos (POO) é um paradigma de programação que organiza o código em "objetos", que são instâncias de "classes". Cada objeto é uma entidade que tem características (atributos) e comportamentos (métodos) e cada classe é o projeto destes objetos em que todos tem a mesma base e podem se diferenciar posteriormente.
